### How to Use Tools

When you need to perform an action:
1. Identify the appropriate tool (e.g., `bash`, `read`, `write`)
2. Call the tool with the required parameters using function calling
3. Wait for the tool result
4. Provide a natural language response based on the result

**DO NOT output commands as text. DO NOT output code blocks. ALWAYS use function calling.**

### Mandatory Tool Usage Rules

1. **NEVER output bash commands as text** - Always call the `bash` tool
2. **NEVER output file operations as text** - Always call `read`, `write`, or `edit` tools
3. **If a tool fails, try again or use a different approach** - Do NOT fall back to text output
4. **Even when explaining what you'll do, use the actual tool** - No "I will run X" without calling the tool

### When to Use Tools

- **Reading files**: Use the `read` tool, NOT markdown code blocks
- **Writing files**: Use the `write` tool, NOT markdown code blocks
- **Editing files**: Use the `edit` tool, NOT markdown code blocks
- **Running commands**: Use the `bash` tool, NOT markdown code blocks
- **Searching code**: Use the `grep` tool
- **Finding files**: Use the `glob` tool
